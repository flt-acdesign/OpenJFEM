LUDWIG Post-Processor

Interactive HTML5/Babylon.js 3D viewer for OpenJFEM structural results with dual-model comparison

Quick Start Interface Struct Tab Vis Tab Aero Tab Navigation Model Comparison Color Scale Element Display

Quick Start

1 Open postv11.html in any modern browser (Chrome, Edge, Firefox)
2 Click the Struct tab
3 Load a .jfem file using the Model A file input
4 The model appears in the 3D viewport with color-coded results
5 Use mouse to orbit, zoom, and pan (see Navigation)
No server or installation required. Just open the HTML file directly in your browser. The only dependency (Babylon.js) is loaded from a CDN.
The .jfem file must be generated by OpenJFEM. Run the solver first: julia -t auto --project=. src/main.jl models/fwing/fwing.bdf

Struct Tab

The primary tab for structural analysis visualization.

Loading Files

Two file inputs are provided for dual-model comparison:

  • Model A — Primary model (cyan color label)
  • Model B — Secondary model for comparison (orange color label)

Both accept .jfem binary files. You can load one or both models. When only one model is loaded, the viewer works identically to a single-model viewer.

Model Units

Select the coordinate system units of the BDF model:

  • mm (default) — coordinates in millimeters, scaled to meters for display
  • m — coordinates already in meters

Subcases

Each model has its own subcase selector (labeled A and B). Select which load case to visualize for each model independently.

Color Scale

Controls the mapping from result values to colors.

  • Shell Range — Min/Max values for shell elements
  • Bar Range — Min/Max values for bar elements
  • Auto Scale — automatically fits range to data (recommended)
  • Symmetric (±max) — centers the scale around zero, useful for signed quantities like bending moments

Deformation

Controls how displacements are amplified for visualization.

  • Scale slider — drag to adjust deformation amplification
  • Value — enter an exact scale factor
  • Auto Scale — sets scale so max displacement is ~10% of model size
Set deformation scale to 0 to see the undeformed shape. Turn on "Undeformed Ghost" to overlay the original shape as a transparent wireframe.

Vis Tab

The Visualization tab provides controls for dual-model comparison and global display settings.

Vertical Offset

When comparing two models, the vertical offset slider separates them along the Z-axis for side-by-side viewing. The offset is proportional to the model size.

  • 0 — models overlaid (same position)
  • 0.5 — models separated by roughly one model height
  • 1.0 — maximum separation (2x model diagonal)

Per-Model Controls

Model A

  • Visible — toggle Model A visibility
  • Transparency — 0% (fully opaque) to 100% (fully transparent)

Model B

  • Visible — toggle Model B visibility
  • Transparency — 0% (fully opaque) to 100% (fully transparent)

Display

  • Show Axes — toggle the 3D coordinate axes indicator (synced with global axes checkbox)
  • Show Model Labels — floating labels showing filename for each model, projected onto the viewport

Dual-Model Comparison

LUDWIG supports loading two JFEM files simultaneously for visual comparison. This is especially useful for comparing OpenJFEM results against MYSTRAN reference solutions.

Typical Comparison Workflow

1 Run OpenJFEM on your model to get model.jfem
2 Run the MYSTRAN converter to get model.MYSTRAN.jfem
3 Open LUDWIG and switch to the Struct tab
4 Load model.jfem into Model A
5 Load model.MYSTRAN.jfem into Model B
6 Switch to the Vis tab
7 Increase the Vertical Offset slider to separate the models
8 Adjust transparency to see through one model to the other
9 Compare deformed shapes, stress patterns, and boundary conditions
Both models share the same color scale and deformation amplification, so colors and deformed shapes are directly comparable. The auto-scale scans both models to set a unified range.

Comparison Tips

  • Overlay mode — set offset to 0, make one model semi-transparent, to see shape differences directly
  • Side-by-side — increase offset, keep both opaque, for color pattern comparison
  • Toggle visibility — quickly switch between models using the Vis tab checkboxes
  • Independent subcases — select different subcases for each model if needed

Color Scale

The color bar at the bottom of the viewport shows the current mapping from result values to colors. It uses a jet-like palette (blue → cyan → green → yellow → red).

Scale Modes

  • Auto Scale — fits to data range (scans both models)
  • Manual — uncheck Auto Scale and enter custom Min/Max values
  • Symmetric — centers around zero (±max), useful for signed quantities

The color bar title shows the active field name. Labels show min and max values.

Status Bar

The status bar at the bottom of the control panel shows a summary of loaded data:

  • A: Model A filename, node count, element breakdown (Q=quads, T=trias, B=bars, R=rods, S=springs, RBE2, RBE3)
  • B: Same for Model B (if loaded)
  • Surface/Net/Nodal: Aero data counts (if loaded)

Supported File Format

LUDWIG reads .jfem files (JFEM Binary Format v3), produced by the OpenJFEM solver and the MYSTRAN converter tool. The format contains:

  • Node coordinates and element connectivity
  • Constraint definitions (SPC, CELAS, RBE2, RBE3)
  • Per-subcase results: displacements, shell forces/strains, bar forces, rod forces
  • Applied forces and moments

The binary format uses little-endian encoding with a magic header (JFEM) and is fully documented in src/Export.jl.
